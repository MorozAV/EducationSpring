## ОПИСАНИЕ ПРОЕКТА: DDL History Monitor

### Информация о проекте
- **Название**: EducationSpring
- **Репозиторий**: https://github.com/MorozAV/EducationSpring.git
- **Предметная область**: Мониторинг и анализ DDL операций в Oracle БД через таблицу `moroz.DDL_HIST`

### Структура таблицы
```sql
create table moroz.DDL_HIST (
    ID                NUMBER generated as identity primary key,
    OBJECT_TYPE       VARCHAR2(4000),
    OWNER             VARCHAR2(4000),
    OBJECT_NAME       VARCHAR2(4000),
    USER_NAME         VARCHAR2(4000),
    DDL_DATE          DATE,
    DDL_TYPE          VARCHAR2(4000),
    CLIENT_OS_USER    VARCHAR2(4000),
    DDL_TXT           CLOB,
    STACK             VARCHAR2(4000),
    CLIENT_IP_ADDRESS VARCHAR2(50),
    CLIENT_HOST       VARCHAR2(4000),
    CLIENT_MODULE     VARCHAR2(4000)
);
```

### Основной функционал
1. **UI (Vaadin 8 с MVP паттерном)**:
    - Отображение списка DDL операций + пагинация
    - Фильтрация по типам операций (ComboBox)
    - Фильтрация по исполнителям (TextField)
    - Просмотр деталей операции (Window)
    - Классифицировать запись вручную (выбрать тег и записать его)
    - Удалить запись

2. **Классификация операций** (Job):
    - **Rule-based classifier** (система правил)
    - **CRITICAL**: DROP TABLE, TRUNCATE, UPDATE PACKAGE
    - **MAINTENANCE**: CREATE INDEX, ANALYZE
    - **TEMP**: временные объекты, GRANT

3. **Очистка** (Job):
    - Удаление записей старше 3 месяцев

4. **Сравнение подходов к БД**:
    - Реализовать одну и ту же функциональность используя:
        - Чистый JDBC
        - Spring Data JPA
        - Spring Data JDBC
        - JdbcTemplate
        - MyBatis
        - JOOQ

### Архитектура
- **Многослойный монолит**: Controller → Service → Repository → Entity
- **Планировщики**: сравнить Spring Scheduler и Quartz

## ДОПОЛНИТЕЛЬНЫЕ ПРАВИЛА

1. **Размер заданий**: Строго 10-50 строк кода или 10-30 минут работы
2. **TDD подход**:
    - Добавляй тесты гибридно - там где это необходимо для понимания
    - **Unit тесты**: используй mock'и (Mockito, MockMvc)
    - **Integration тесты**: предоставляй SQL скрипты для подготовки тестовых данных в Oracle, я запущу их вручную
3. **Параллельное изучение**: Сравнивай технологии в одном проекте (например, один CRUD через JPA, другой через JDBC)
4. **Постепенность**: От простого к сложному
5. **Практичность**: Каждое задание должно давать работающий код
6. **Обратная связь**: Всегда объясняй "почему" при проверке
7. **Vaadin + Spring**: В Vaadin 8 добавляем Spring интеграцию постепенно, по мере развития его поддержки в версиях Vaadin (не обязательно использовать Spring полностью на начальном этапе)
8. **Версия Java**: Повышать версию Java постепенно, в зависимости от минимальных требований текущей версии Vaadin и Spring. Начинаем с Java 8, переходим на более новые версии только когда это необходимо для совместимости с фреймворками
